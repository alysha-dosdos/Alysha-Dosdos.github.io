Reflect on how confident you feel using each of these problem-solving techniques/processes:

  • Pseudocode- I think it is good to have just to put my initial thought of how I think it's going to work and then just slowly build on it. I just need to have my pseudocode
  • Trying something- this is pretty much as the one above, just blindly testing to see if it works can eventually lead to the right solution but it just takes a lot of time with trial and error.
  • Rubber ducky method- I'm confident I can use this as long as I know how each part of the code works but if I don't even know what I'm doing in the first place, then it is useless trying to explain. It does help me realize what each part is supposed to be doing and figuring out what knowledge I lack too.
  • Reading error messages- It does have some useful hints but it makes it difficult when I don't know what I should be using instead. I just end up being lost not knowing how to fix it.
  • Console.logging- this sometimes just straight up doesn't even work while I was doing Kata but I knew that when it doesn't come up with anything just means that it's probably undefined and I'm probably doing something wrong anyway. It is useful to figure out if each part of my code is working as intended.
  • Googling- This was a bit difficult to use at first because I was overwhelmed with all the possible answers and didn't even know where to start. There were just too many to choose from and they were all in different context.
  • Asking your peers for help- I was lucky to have a friend who was studying with me in the course we were both in the same boat and could help give each other hints on how to make things work. I'm also lucky to have friends in tech who already has experience in tech and can help out which is nice.
  • Asking coaches for help- I always try to ask for help in discord first before I ask anyone else. The only problem is time because I have to make sure to ask before 5pm but sometimes I get work done after 5pm so there's no one free to ask in the facilitators.
  • Improving your process with reflection- This is good to recap what I just learnt and to try and really understand why the code is working. The only problem I have so far is that I don't have as much time as I'd like to really delve deep into understanding each section.
